# Spoonjoy Storybook

This Storybook is intentionally curated. It documents current Spoonjoy product surfaces and the small shared foundation those surfaces actually use.

If a story does not represent something visible in today's app, support a current app flow, or explain a reusable foundation pattern in context, it does not belong here. Exhaustive primitive inventories, old design-token catalogs, internal dock dissections, and test-only interaction fixtures live better in code, tests, or not at all.

## What Stays Here

- **App/Foundation** — A compact snapshot of current UI primitives as Spoonjoy uses them.
- **Brand/SpoonjoyLogo** — The active brand mark and real app placements.
- **Navigation/MobileNav** — Current mobile information architecture: New, Home, List, and logged-out Home/Login.
- **Components/ConfirmationDialog** — The confirmation pattern used for destructive app actions.
- **Pantry** — The current pantry/home recipe surfaces.
- **Recipe/Input/RecipeBuilder** and **Recipe/View/RecipeView** — Current high-level recipe creation and reading experiences.

## What Does Not Belong Here

- Stories for unused or internal-only implementation pieces.
- Exhaustive prop permutations that duplicate component APIs without product context.
- Test-only stories whose job is covered by Vitest or route/component tests.
- Stale examples for old navigation, old button props, or retired design tokens.

## Commands

```bash
pnpm storybook
pnpm build-storybook
pnpm test -- --run test/storybook-sync.test.ts
```

Treat this as Spoonjoy's visual field guide: enough to understand the app quickly, not so much that the signal gets buried under pantry clutter.
